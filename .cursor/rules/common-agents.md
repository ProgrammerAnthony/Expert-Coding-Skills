# Agent 编排

## 按场景选用技能

| 场景 | 推荐技能（斜杠命令） | 说明 |
|------|---------------------|------|
| 实施前规划、方案对比 | 方案设计师 `/方案设计` | 澄清需求、对比方案、输出设计文档 |
| 架构与技术方案 | 架构顾问 `/架构分析` | 新系统设计或现有架构优化 |
| 测试驱动开发 | TDD 开发大师 `/tdd` | 新功能与 Bug 修复 |
| 通用代码审查 | 代码审查专家 `/代码审查` | 修改代码后的质量检查 |
| 安全与白盒审计 | 代码安全审计专家 `/代码安全审计` | 提交前或敏感模块 |
| 官方文档与 API | 文档查询 | 查库/框架用法，减少幻觉 |
| 问题排查与根因 | 调试专家 `/调试` | 异常、失败测试、行为不符预期 |
| 前端专项 | 前端代码审查 `/前端代码审查` 等 | React/Vue/Next 等；其他前端技能见项目技能清单 |

端到端测试、大规模文档更新、构建错误集中分析等未单独列出的工作，可组合使用 **调试专家**、**代码审查专家** 等技能分步完成；相互独立的子问题应 **并行** 调度（见下文）。

## 应主动选用的场景

无需用户额外说明也应主动选用：

1. **复杂功能需求** — **方案设计师** `/方案设计`
2. **刚写完或刚改完代码** — **代码审查专家** `/代码审查`
3. **Bug 修复或新功能** — **TDD 开发大师** `/tdd`
4. **架构类决策** — **架构顾问** `/架构分析`

## 并行执行任务

对相互独立的操作，**务必**使用并行 Task / 多 Agent 并行：

```markdown
# 推荐：并行执行
同时启动 3 个 Agent：
1. Agent 1：认证模块安全分析
2. Agent 2：缓存系统性能审查
3. Agent 3：工具函数类型检查

# 不推荐：无必要地串行
先跑完 Agent 1，再跑 Agent 2，再跑 Agent 3
```

## 多视角分析

对复杂问题，可拆分子角色分别从以下角度审查：

- 事实与依据核对
- 资深工程师视角
- 安全专家视角
- 一致性与规范
- 冗余与重复
