# 通用模式

## 脚手架项目

实现新功能时：

1. 搜索经过实战检验的脚手架/模板项目
2. 使用并行 Agent 评估候选方案：
   - 安全评估
   - 可扩展性分析
   - 与需求的匹配度打分
   - 实施计划
3. 以最佳匹配为起点克隆或借鉴
4. 在已验证的结构上迭代

## 设计模式

### Repository（仓储）模式

将数据访问封装在统一接口之后：

- 定义标准操作：`findAll`、`findById`、`create`、`update`、`delete`
- 具体实现负责存储细节（数据库、API、文件等）
- 业务逻辑依赖抽象接口，而非具体存储方式
- 便于替换数据源，也便于用 mock 做测试

### API 响应格式

对所有 API 响应使用一致的外层结构：

- 包含成功/状态标识
- 包含数据载荷（错误时可为 null）
- 包含错误信息字段（成功时可为 null）
- 分页响应包含元数据（总数、页码、每页条数等）
