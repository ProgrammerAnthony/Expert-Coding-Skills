# 代码审查专家

资深工程师视角的结构化代码审查技能，覆盖 SOLID 原则、安全漏洞、性能问题、边界条件与可删除代码识别。

## 安装

```bash
npx skills add <username>/expert-coding-skills --path skills/code-review-expert
```

## 使用方式

在 Agent 终端中输入：

```
/代码审查
```

或：

```
/code-review
```

技能将自动获取当前 git 变更，并进行全面的结构化审查。

## 审查维度

| 维度 | 内容 |
|------|------|
| SOLID 原则 | 单一职责、开闭、里式替换、接口隔离、依赖倒置 |
| 安全漏洞 | 注入、认证、授权、文件操作、SSRF、加密、敏感信息泄露 |
| 代码质量 | 错误处理、性能热点、边界条件、可测试性 |
| 可删除代码 | 无用代码、冗余逻辑、死代码识别 |

## 严重度分级

- **P0 致命**：必须阻止合并
- **P1 严重**：合并前应修复
- **P2 中等**：本 PR 修复或创建后续 Issue
- **P3 建议**：可选改进

## 输出示例

```markdown
## 代码审查报告

### 总览
本次变更涉及用户认证模块，新增 JWT 刷新逻辑，修改 3 个文件，增加 127 行。

### 发现问题

#### P0 致命问题
- **[src/auth/jwt.py:45]** JWT 密钥硬编码在源码中
  - 原因：密钥泄露后无法轮换，攻击者可伪造任意 token
  - 修复建议：从环境变量读取，`secret = os.environ["JWT_SECRET"]`

#### P1 严重问题
- **[src/auth/handler.py:88]** refresh_token 未校验过期时间
  ...

### 亮点
- token 生成使用了 `secrets.token_urlsafe()`，安全性良好
```

## 注意事项

- 技能默认只输出审查报告，不自动修改代码
- 用户需明确确认后才会执行修复操作
