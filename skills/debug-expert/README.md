# 调试专家

系统化调试工作流，采用 4 阶段根因分析法，禁止猜测式修改，完成前强制验证。

## 安装

```bash
npx skills add <username>/expert-coding-skills --path skills/debug-expert
```

## 使用方式

```
/调试
```

或：

```
/debug
```

## 调试流程

```
问题理解（收集完整错误信息）
  ↓
建立假设（2-5 个排序后的假设）
  ↓
最小复现（建立稳定可重复的测试案例）
  ↓
定位根因（二分法缩小范围）
  ↓
修复方案（多方案对比选最优）
  ↓
完成验证（强制检查清单）
```

## 核心原则

1. **完整错误信息优先**：不看完整堆栈，不开始分析
2. **假设驱动**：先列假设再验证，而不是随机修改
3. **最小复现**：建立可以稳定触发问题的最小场景
4. **根因而非症状**：不做表面修复掩盖真实问题
5. **完成前验证**：修复后必须运行验证清单

## 擅长的问题类型

- 异常和堆栈跟踪分析
- 竞态条件和并发问题
- 环境差异导致的问题
- 性能问题分析
- 内存泄漏排查
- 数据库查询问题
