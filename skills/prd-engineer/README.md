# 需求工程师

通过用户访谈和代码库探索，编写高质量 PRD，并按需拆解为可执行的 GitHub Issues 和实施计划。

## 安装

```bash
npx skills add <username>/expert-coding-skills --path skills/prd-engineer
```

## 使用方式

```
/写PRD
```

或：

```
/prd
```

## 三种工作模式

| 模式 | 输出 | 适用场景 |
|------|------|----------|
| 仅写 PRD | 产品需求文档 | 需要与团队对齐需求 |
| PRD + Issues | 文档 + GitHub Issues | 需要进入开发排期 |
| 完整模式 | 文档 + Issues + 实施计划 | 大型功能，需要详细规划 |

## 工作流

```
初始问卷（了解背景和痛点）
  ↓
代码库探索（了解现有实现）
  ↓
迭代访谈（苏格拉底式追问）
  ↓
PRD 编写（分节展示，逐节确认）
  ↓
[可选] Issues 拆解（每个 2-5 天工作量）
  ↓
[可选] 实施计划（依赖关系 + 迭代划分）
```

## PRD 包含内容

- **问题陈述**：用户的真实痛点
- **解决方案**：高层次描述（不写代码）
- **用户故事**：具体场景 + 边界情况
- **验收标准**：可测试的通过条件
- **Out of Scope**：明确不做什么

## Issues 示例

```markdown
## feat: 用户注册邮箱验证

**用户故事**：
作为新用户，我想要在注册后收到验证邮件，以便确认我的邮箱真实有效。

**验收标准**：
- [ ] 注册成功后 30 秒内收到验证邮件
- [ ] 验证链接有效期 24 小时
- [ ] 点击链接后账号状态变为"已验证"
- [ ] 验证链接过期后提示重新发送

**技术决策**（留给开发者）：
- 邮件服务选择
- Token 存储方案

**估算**：3 天
```
